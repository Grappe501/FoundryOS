# Scope Doctrine

Protect the core vision from scope drift.

## The Risk

You can build forever:

```txt
Knowledge · Mastery · Projects · Community · Identity · Legacy · Mentorship · Purpose · Outcomes
```

The danger is becoming **everything to everyone**.

## The Opportunity

Become the **infrastructure** that powers all of those things — not each layer as a standalone product forever.

## The Rule

> Build transformation systems — not isolated features.

## PASS-009

Don't think about generating entities. Think about generating **Transformation Systems**.

Every entity must eventually connect to:

- Outcomes
- Paths
- Projects
- Communities
- Mentors
- Roles
- Legacy

An entity by itself is just information. An entity connected to a transformation system becomes part of a journey.

## Pass Gate

Before adding a feature, ask: does this strengthen **Potential → Capability → Agency → Contribution → Legacy**, or does it expand surface area without deepening transformation?

Does it increase **agency** — can the person take meaningful action?
