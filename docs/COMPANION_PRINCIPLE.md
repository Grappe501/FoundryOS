# Companion Principle

Foundry is **not a destination**. It is a **companion**.

## Platforms vs Foundry

| Platform | You visit to… |
|----------|---------------|
| Wikipedia | look something up |
| YouTube | watch something |
| Reddit | discuss something |
| MasterClass | learn something |
| **Foundry** | **become something** |

That's fundamentally different.

## Doctrine

Every major system must answer:

> **How does this help the user today?**

Not:

> How does this improve the platform?

## Home Screen

Should not feel like software. Should feel like:

```txt
Here's where you are.
Here's what's working.
Here's what's next.
Here's why it matters.
```

## Daily Loop

| Rhythm | Question |
|--------|----------|
| **Morning** | What am I becoming? |
| **During the Day** | What should I work on? |
| **Evening** | What did I learn? |
| **Long-Term** | How have I changed? |

PASS-010 proves the intelligence loop. The daily experience is how a companion shows up.

## Long-Term Test

Twenty years from now, the most successful user shouldn't say:

```txt
I learned a lot.
```

They should say:

```txt
I became the person I wanted to become.
```
