# Transformation Operating System

Foundry is not a knowledge platform. It is a **Transformation Operating System**.

Foundry optimizes for **agency** — the bridge between knowing and doing.

---

## Four Object Categories

---

## Aspirations

What someone is becoming.

```txt
Become a Leader
Become a Doctor
Become a Gardener
Become a Programmer
Become a Historian
```

Maps to: outcomes, human outcomes registry, user path selections.

The machinery of change.


Maps to: `@foundry/outcome-engine`, `@foundry/path-engine`, `@foundry/project-engine`, domain DNA records.

## Evidence

Proof that transformation happened.

```txt
Projects Completed
Artifacts Created
Reviews Written
Mentoring Provided
Communities Built
```

Maps to: `@foundry/evidence-engine`, evidence profiles, evidence tiers (PASS-011).

---

## Legacy

What remains after transformation.

```txt
People Helped
Communities Impacted

```

Maps to: legacy engine, journey events, mentorship outcomes.

## The Ultimate Questions

1. **What should I do next?** — Next Highest-Value Action Engine
2. **What did you learn?** — Reflection Engine
3. **Did it help?** — Transformation feedback loop
4. **Is this person moving forward?** — Transformation Momentum Engine

If the graph answers these accurately, Foundry becomes a guide for human growth — not a repository of knowledge.
